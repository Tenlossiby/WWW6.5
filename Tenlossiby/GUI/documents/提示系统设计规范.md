# 提示系统设计规范

> **版本**: v1.0
> **更新日期**: 2026-03-13
> **适用范围**: Day1 - Day11+

---

## 📋 设计原则

### 核心理念：**不怕重复，就怕初学者不知道怎么操作**

- ✅ **双重提示** - 重要信息会在多个地方显示
- ✅ **即时反馈** - 每个操作都有明确的响应
- ✅ **持续引导** - 用户随时能看到"下一步做什么"
- ✅ **容错设计** - 即使用户错过某个提示，也能在其他地方看到

---

## 🎯 提示系统架构

### 提示渠道对比

| 维度 | 小弹窗 (Toast) | 交互提示块 (Knowledge Panel) |
|------|---------------|------------------------------|
| **显示时长** | 临时（3秒后自动消失） | 持久（一直显示直到状态改变） |
| **内容类型** | 操作确认 + 下一步提示 | 下一步引导 / 知识点详细解释 |
| **触发时机** | 操作后立即显示 | 基于当前解锁状态自动计算 |
| **显示位置** | 页面底部中央 | 右侧知识面板 |
| **优势** | 即时性、醒目、不易错过 | 可回顾、详细、持续可见 |
| **劣势** | 一闪而过、可能被忽略 | 需要用户主动查看侧边栏 |

## 📐 提示文本编写规范

### 1. 小弹窗 (Toast Message)

#### 用途
- **操作确认** - 告知用户操作结果（成功/失败）
- **即时反馈** - 显示当前状态变化
- **下一步引导** - 提示用户接下来应该做什么

#### 文本结构

**成功操作：**
```
✅ 操作确认！👉 下一步引导
```

**失败操作：**
```
❌ 错误原因
```

**解锁新概念：**
```
✅ 操作成功！
🎉 恭喜解锁：[概念名称]！👉 下一步引导
```

#### 示例

**切换身份：**
```javascript
// 切换到用户身份
'✅ 已切换到用户身份！👉 现在尝试提取资金来体验权限控制！'

// 切换到所有者身份
'✅ 已切换到所有者身份！👉 以所有者身份可以转移所有权和提取资金！'
```

**存入 ETH：**
```javascript
// 首次存款（解锁概念）
'✅ 存款成功！📥 恭喜解锁：导入语句！👉 继续存入 ETH 来学习事件日志！'

// 非首次存款
'✅ 存款成功！'
```

**查询操作：**
```javascript
// 查看所有者
'👤 当前所有者：0x1234...5678 📦 恭喜解锁：合约继承！👉 存入 ETH 来学习 import 机制！'

// 查看余额
'📊 合约余额：1.0000 ETH'
```

**错误提示：**
```javascript
// 权限不足
'❌ 权限不足：只有所有者才能提取资金 🛡️ 解锁知识点：onlyOwner 修饰符！'

// 参数错误
'❌ 请输入有效的存入数量！'
```

### 2. 交互提示块 (Knowledge Panel Hint Text)

#### 用途
- **下一步引导** - 显示当前概念解锁后的下一步操作
- **知识点解释** - 用户点击概念标签后显示详细解释
- **未解锁概念提示** - 显示剩余概念的解锁指引

#### 文本结构

**下一步提示（自动显示）：**
```
🎉 恭喜解锁：[概念名称]！👉 下一步具体操作来学习 [下一个概念]！
```

**知识点解释（手动点击后）：**
```
📖 [概念名称]：[详细解释，包括技术原理、使用场景、注意事项]
```

**未解锁概念指引：**
```
👉 具体操作来学习 [概念名称]！
```

#### 示例

**下一步提示：**
```javascript
// 解锁 inheritance 后
'📦 太棒了！你看到 VaultMaster 继承了 Ownable 的功能！👉 存入 ETH 来学习 import 机制！'

// 解锁 constructor 后
'🏗️ 太棒了！你了解了构造函数！👉 存入 ETH 来学习导入语句和私有变量！'

// 解锁 import_statement 后
'📥 不错！你了解了导入语句！👉 继续存入 ETH 来学习事件日志和私有变量！'

// 解锁 private_visibility 后
'🔒 优秀！你学会了 private 变量的使用！👉 尝试转移所有权来学习所有权机制！'
```

**知识点解释：**
```javascript
// 点击 constructor 标签
'📖 构造函数 `constructor()` 在合约部署时自动执行一次，用于初始化合约的状态变量。在 Ownable 中，它将合约部署者设置为初始所有者。'

// 点击 private_visibility 标签
'📖 `private` 可见性修饰符表示变量只能在当前合约内部访问，即使是子合约也无法直接访问。这提供了最强的封装性，保护敏感数据。'
```

**未解锁概念指引：**
```javascript
// remainingConceptsGuide
'👉 切换到"用户"身份，尝试提取资金，体验权限控制！
👉 以所有者身份转移所有权，学习所有权机制！
👉 存入 ETH 来触发事件，了解导入和事件日志！'
```

## 🔄 双重提示逻辑

### 为什么要重复显示？

1. **容错设计** - 用户可能错过小弹窗（3秒后消失）
2. **强调重点** - 重要引导信息重复显示能加深印象
3. **不同场景** - 小弹窗适合即时反馈，侧边栏适合持续引导

### 重复示例

**场景：切换到用户身份**

```
小弹窗（3秒）：
✅ 已切换到用户身份！👉 现在尝试提取资金来体验权限控制！

交互提示块（持久）：
👉 切换到"用户"身份，尝试提取资金，体验权限控制！
```

**说明：** 两处文本略有不同，但核心信息一致。小弹窗更即时，侧边栏更持久。

## 📝 实施规范

### Composable 返回值结构

```javascript
// 操作成功，有新概念解锁
return {
  success: true,
  message: '✅ 操作确认！🎉 恭喜解锁：[概念]！👉 下一步操作',
  hints: ['🎉 恭喜解锁：[概念A]！👉 下一步操作', '🎉 恭喜解锁：[概念B]！']
}

// 操作成功，无新概念解锁
return {
  success: true,
  message: '✅ 操作确认！',
  hints: []
}

// 操作失败
return {
  success: false,
  error: '❌ 错误原因',
  hint: '🛡️ 解锁知识点：[概念]！👉 下一步操作'
}
```

---

### 组件处理逻辑

```javascript
// 显示成功消息
const handleSuccess = (result) => {
  if (result.message) {
    showMessage(result.message, false)  // 小弹窗
  }
  // hints 会自动更新到交互提示块，无需手动处理
}

// 显示错误消息
const handleError = (result) => {
  if (result.error) {
    showMessage(result.error, true)  // 小弹窗，红色
  }
  if (result.hint) {
    showMessage(result.hint, false)  // 小弹窗，普通颜色
  }
}
```

---

### KnowledgePanel 显示优先级

```javascript
const hintText = computed(() => {
  // 优先级 1: Day 11 显示最新解锁概念的下一步提示
  if (props.currentDay === 11 && currentConcept.value && !manualConceptKey.value) {
    return getNextStepHint(currentConcept.value.key)
  }

  // 优先级 2: 用户手动点击概念标签，显示详细解释
  if (manualConceptKey.value && currentConcept.value) {
    return getConceptHint(currentConcept.value.key)
  }

  // 优先级 3: 外部传入的自定义提示
  if (props.customHint) {
    return props.customHint
  }

  // 优先级 4: 显示当前概念的详细解释（默认）
  return currentConcept.value ? getConceptHint(currentConcept.value.key) : ''
})
```

## 🎯 总结

**核心原则：**
- 双重提示，确保信息传达
- 即时反馈 + 持续引导
- 不怕重复，怕不知道

**实施要点：**
- 小弹窗：操作确认 + 解锁提示 + 下一步引导
- 交互提示块：下一步提示 / 知识解释 / 未解锁指引
- 两者配合，确保初学者不会迷失

**适用范围：**
- Day1 - Day11+ 所有交互式学习内容
- 所有需要引导用户的操作流程
- 所有概念解锁机制
